2. ✅ Checklist ครั้งแรกหลังอัปเดตระบบ (สำคัญ — ต้องทำ!)

ระบบสิทธิ์เปลี่ยนใหม่: ตอนนี้การเขียนข้อมูล (เพิ่ม/แก้/ลบ) ทำได้เฉพาะบัญชีที่มี role admin เท่านั้น และ role จะติดมากับ token หลังล็อกอินใหม่
1
เปิด Supabase: cd supabase/docker แล้ว docker compose up -d (เช็คว่า container supabase-kong รันอยู่ที่ port 8000 ด้วย docker ps)
2
รันเว็บ: npm run dev → เปิด http://localhost:8080
3
ถ้าเคยล็อกอินค้างไว้ → กด Logout ก่อน แล้ว Login ใหม่ ที่ http://localhost:8080/login ด้วย admin@dslab.com (ถ้าไม่ทำ จะกดบันทึกแล้วขึ้น "Unauthorized")
4
ทดสอบระบบ 1 รอบ (ส่วนนี้ผมทดสอบแทนไม่ได้ ต้องทำในเบราว์เซอร์จริง):
  • ไปหน้า Software → กดปุ่ม Add → กรอกชื่อ TH/EN → Save → ต้องขึ้น "Success"
  • กลับมาหน้า Software → ต้องเห็นโปรเจกต์ใหม่แสดงผล
  • ลองอัปโหลดรูปในฟอร์ม → ต้องอัปสำเร็จและรูปแสดง (ระบบ Storage เพิ่งถูกตั้งสิทธิ์ใหม่)
  • เปิดหน้าเว็บแบบ "ไม่ล็อกอิน" (Incognito) → ต้องเห็นข้อมูลปกติ แต่เข้า /admin/... ไม่ได้

3. การเข้าสู่ระบบ (Login)

1
เข้า http://localhost:8080/login
2
กรอกอีเมล admin@dslab.com + รหัสผ่าน
3
ล็อกอินสำเร็จ → ปุ่ม Admin (เพิ่ม/แก้ไข) จะปรากฏบนหน้าเว็บ และเข้าเมนู ตั้งค่าบริษัท ได้จาก Navigation
บัญชีที่สมัครใหม่ทั่วไป ไม่ใช่ admin — ล็อกอินได้แต่เข้า /admin ไม่ได้ และเขียนข้อมูลไม่ได้ (ถูกบล็อกทั้งระดับหน้าจอและระดับฐานข้อมูล)

วิธีเพิ่มบัญชี Admin คนใหม่

1) ให้คนนั้นสมัคร/สร้างบัญชีก่อน แล้วรันคำสั่งนี้ (เปลี่ยนอีเมล):

docker exec supabase-db psql -U postgres -d postgres -c \
"UPDATE auth.users SET raw_app_meta_data = COALESCE(raw_app_meta_data,'{}'::jsonb) || '{\"role\":\"admin\"}'::jsonb WHERE email='new-admin@example.com';"

2) ให้บัญชีนั้น logout → login ใหม่ role จึงจะมีผล

5. ระบบความปลอดภัยที่ตั้งไว้ (สรุปสำหรับอ้างอิง)

  • DB ทุกตารางเปิด RLS: คนทั่วไปอ่านได้อย่างเดียว — เพิ่ม/แก้/ลบได้เฉพาะ admin (ตรวจจาก token จริง ปลอมจากฝั่งเบราว์เซอร์ไม่ได้)
  • Storage รูป/ไฟล์ใน bucket images, landpages-media: ทุกคนเปิดดูได้ แต่อัปโหลด/ลบได้เฉพาะ admin
  • หน้าเว็บ เส้นทาง /admin/* ทั้งหมดเช็ค role admin — ไม่ใช่ admin จะถูกเด้งไปหน้า login
  • ไฟล์ SQL การตั้งค่าทั้งหมดบันทึกไว้ใน rls_policies.sql และ migrate_to_flat_schema.sql (รัน DB ใหม่เมื่อไหร่ ใช้ 2 ไฟล์นี้ apply ซ้ำได้)

6. ปัญหาที่อาจเจอ + วิธีแก้

อาการสาเหตุ / วิธีแก้
กด Save แล้วขึ้น "Unauthorized" Token ยังไม่มี role admin → Logout แล้ว Login ใหม่
อัปโหลดรูปไม่ได้ เช็คว่าล็อกอินเป็น admin + ไฟล์เป็นรูป (jpg/png/webp) ขนาดไม่เกิน 5MB
หน้าเว็บขึ้นว่าง / โหลดไม่จบ Supabase ไม่ได้รัน → docker compose up -d ใน supabase/docker แล้วรีเฟรช
ลืมรหัสผ่าน admin Reset ผ่าน Supabase Studio (http://localhost:8000 → Authentication → Users)
หมายเหตุ: ลิงก์อีเมล reset อาจชี้ไป localhost:3000 — ถ้าจะใช้ flow อีเมลให้แก้ SITE_URL ใน supabase/docker/.env เป็น http://localhost:8080 แล้ว restart docker
เพิ่มหมวดหมู่ Software ใหม่ใน dropdown หมวดอยู่ในตาราง software_categories — เพิ่มผ่าน Studio หรือ:
docker exec supabase-db psql -U postgres -d postgres -c "INSERT INTO software_categories (name) VALUES ('ชื่อหมวด');"
เคล็ดลับ: ข้อมูลเก่าที่ย้ายมาจากระบบเดิม ช่องภาษาไทยบางช่องถูกเติมด้วยข้อความอังกฤษไว้ชั่วคราว — เข้าไปกด Edit แต่ละรายการเพื่อใส่ภาษาไทยที่ถูกต้องได้เลย
